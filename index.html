1. Infrastruttura Globale AWS

Mappa globale delle Regioni AWS per garantire bassa latenza, tolleranza ai guasti e sovranità dei dati in tutto il mondo.

Regioni (Regions)

Aree geografiche fisiche sparse per il mondo (es. eu-south-1 a Milano). Ogni regione è completamente indipendente e isolata dalle altre per garantire la massima tolleranza ai guasti.

Come scegliere una Regione per l'esame:

  • Latenza (vicinanza agli utenti finali)
  • Costi (variano per regione)
  • Servizi disponibili (non tutti i servizi sono ovunque)
  • Conformità legale locale (es. GDPR)

Zone di Disponibilità (AZs)

Ogni Regione contiene almeno 2 (spesso 3 o più) Zone di Disponibilità. Un'AZ è costituita da uno o più data center fisici separati (spesso a decine di km di distanza), con alimentazione e rete indipendenti per prevenire guasti a cascata.

Edge Locations & Edge

  • Edge Locations: Punti di presenza (PoP) in tutto il mondo per memorizzare nella cache i contenuti visivi e statici (Usati da Amazon CloudFront / Route 53).
  • Local Zones: Estensioni delle regioni AWS posizionate vicino a grandi centri abitati per latenza a una singola cifra di millisecondi.
  • Outposts: Rack hardware fisici AWS installati direttamente nel tuo data center per creare un cloud ibrido.

2. Sicurezza e Modello di Responsabilità

Il Modello di Responsabilità Condivisa: una delle tematiche in assoluto più richieste all'esame CLF-C02.

Responsabilità di AWS (Sicurezza DEL Cloud)

  • Sicurezza fisica dei Data Center (guardie, telecamere).
  • Infrastruttura di rete globale (cavi, router sottomarini).
  • Hardware fisico, Storage e Server host (Hypervisor).
  • Patching dei servizi gestiti (es. aggiornamenti automatici dell'OS per database RDS o DynamoDB).

Responsabilità del Cliente (Sicurezza NEL Cloud)

  • Gestione e protezione dei dati (Crittografia dei dati a riposo e in transito).
  • Configurazione IAM (Gestione accessi utenti, attivazione MFA, rotazione password).
  • Patching e aggiornamenti del sistema operativo ospite (su cui girano le TUE istanze EC2).
  • Configurazione regole firewall (Security Groups per EC2 e NACL per la VPC).

3. I 50 Servizi AWS Chiave (Cheat Sheet)

💻 Calcolo (Compute) & Container

Amazon EC2Server virtuali nel cloud (IaaS). Capacità ridimensionabile on-demand per ogni sistema operativo.
AWS LambdaEsegue codice in risposta ad eventi senza gestire server (Serverless). Paghi esattamente per i millisecondi usati.
Amazon ECS / EKSServizi per l'orchestrazione e gestione dei container (Docker / Kubernetes).
AWS FargateMotore di calcolo serverless per i tuoi container ECS o EKS. Non gestisci i server sottostanti.
AWS Elastic BeanstalkServizio PaaS: tu carichi solo il codice, AWS gestisce provisioning, load balancing e scaling.
Amazon LightsailVPS preconfigurata, perfetta per progetti semplici o siti web (es. WordPress, Joomla) a basso costo.

💾 Archiviazione (Storage)

Amazon S3Archiviazione di oggetti infinita. Usato per file, immagini, backup, log e hosting di siti web statici.
Amazon EBSVolumi di archiviazione a blocchi. Sono essenzialmente i "dischi rigidi" collegati alle tue istanze EC2.
Amazon EFSFile system di rete condiviso, elastico e serverless per più istanze Linux contemporaneamente.
Amazon S3 GlacierClassi di archiviazione S3 per dati a lungo termine e archivi a freddo a costi estremamente bassi.
AWS Snow FamilyDispositivi fisici (Snowcone, Snowball, Snowmobile) che AWS ti spedisce per migrare enormi quantità di dati offline.

🗄️ Database

Amazon RDSServizio di database relazionali gestito. Supporta MySQL, PostgreSQL, Oracle, SQL Server, MariaDB.
Amazon AuroraDB relazionale proprietario AWS, super performante (compatibile MySQL/PostgreSQL). Fino a 5x più veloce.
Amazon DynamoDBDatabase NoSQL (chiave-valore e documenti) serverless. Offre latenza a singola cifra di millisecondo.
Amazon RedshiftData warehouse cloud progettato per eseguire analisi SQL complesse su petabyte di dati aziendali.
Amazon ElastiCacheServizio di cache in-memory (Redis/Memcached) per accelerare drasticamente le letture dei database.

🌐 Reti e Networking

Amazon VPC(Virtual Private Cloud). Rete virtuale isolata dove lanci le risorse. Il tuo data center logico.
Amazon Route 53Servizio DNS (Domain Name System) cloud scalabile per instradare gli utenti finali ai siti web.
Amazon CloudFrontCDN globale (Content Delivery Network). Usa le Edge Location per distribuire dati a bassissima latenza.
AWS Direct ConnectConnessione di rete dedicata in fibra ottica che collega il tuo ufficio fisico direttamente ad AWS.

🛡️ Sicurezza e Conformità

AWS IAMIl cuore della sicurezza: gestisce Utenti, Gruppi, Ruoli e Policy di accesso scritte in formato JSON.
AWS WAF & ShieldWAF blocca exploit web comuni (es. SQLi, XSS). Shield protegge l'infrastruttura dagli attacchi DDoS.
AWS KMSServizio centralizzato per creare e gestire le chiavi crittografiche per proteggere i tuoi dati.
Amazon GuardDutyAnalizza log di rete usando il Machine Learning per rilevare minacce in tempo reale (Intelligent Threat Detection).
AWS ArtifactIl portale self-service per scaricare i report ufficiali di conformità e sicurezza AWS (ISO, PCI-DSS, SOC).

🛠️ Gestione, Monitoraggio e Governance

Amazon CloudWatchRaccoglie metriche (CPU, rete) e log in tempo reale. Fondamentale per impostare allarmi automatici.
AWS CloudTrailAuditing puro: registra tutte le chiamate API. Risponde alla domanda "Chi ha fatto cosa, quando e da dove?".
AWS CloudFormationInfrastructure as Code (IaC). Crei e replichi interi ambienti scrivendo script o modelli YAML/JSON.
AWS Trusted AdvisorL'esperto cloud automatico. Ti dà consigli su: Costi, Prestazioni, Sicurezza, Tolleranza guasti, Limiti di servizio.
AWS Auto ScalingAggiunge o rimuove automaticamente risorse (come istanze EC2) in base al traffico effettivo del momento.
AWS OrganizationsPermette di raggruppare più account AWS aziendali per ottenere sconti sui volumi e fatturazione unica consolidata.

🧠 Integrazione & Machine Learning

Amazon SQS(Simple Queue Service). Coda di messaggi per disaccoppiare e scalare in modo indipendente i microservizi.
Amazon SNS(Simple Notification Service). Servizio pub/sub per inviare messaggi, email o SMS agli utenti o alle app.
Amazon AthenaFai query SQL direttamente sui file di testo e log archiviati in S3, senza dover creare nessun server database.
Amazon SageMakerPiattaforma completa per sviluppatori e data scientist per addestrare e rilasciare modelli di Machine Learning.

4. AWS Well-Architected Framework

I 6 pilastri fondamentali per progettare architetture cloud di successo.

I 6 Pilastri (The 6 Pillars)

  • Eccellenza Operativa: Eseguire e monitorare i sistemi. Automatizzare modifiche e rispondere agli eventi in modo procedurale.
  • Sicurezza: Proteggere dati, sistemi e risorse tramite crittografia, IAM e tracciabilità.
  • Affidabilità: Capacità di un sistema di riprendersi rapidamente dai guasti, testare il ripristino e garantire ridondanza.
  • Efficienza delle Prestazioni: Scegliere e usare in modo ottimale le risorse IT scalabili rispetto al carico di lavoro.
  • Ottimizzazione dei Costi: Eliminare spese inutili, dimensionare correttamente le istanze (right-sizing) e usare strumenti di pricing.
  • Sostenibilità: Minimizzare l'impatto ambientale e il consumo energetico dell'architettura riducendo gli sprechi.

5. Fatturazione e Costi

Schema concettuale dei metodi di acquisto dell'infrastruttura di calcolo.

Piani tariffari per le Istanze (EC2)

  • On-Demand: Paghi al secondo o all'ora. Nessun impegno. Ideale per carichi imprevedibili e di breve durata.
  • Reserved Instances / Savings Plans: Impegno a priori di 1 o 3 anni. Sconto enorme (fino al 72%). Per carichi di lavoro in esecuzione costante.
  • Spot Instances: Sconto fino al 90%. AWS può spegnerle con 2 minuti di preavviso. Ideali per batch processing e calcoli non critici.
  • Dedicated Hosts: Server fisici dedicati interamente a te. Obbligatorio per regole di compliance legale o licenze software BYOL.

Strumenti Finanziari

  • AWS Pricing Calculator: Da usare prima di lanciare i servizi per fare preventivi.
  • AWS Cost Explorer: Grafici interattivi per analizzare la spesa storica e fare previsioni future.
  • AWS Budgets: Imposta limiti personalizzati e invia allarmi prima che la fattura diventi troppo alta.
  • Piani di Supporto: Basic (gratis), Developer (email), Business (24/7 telefono/chat, accesso completo a Trusted Advisor), Enterprise (Sistemi mission-critical, include TAM).

6. Tips & Tricks per l'Esame

🧠 Cheat Codes: Associazione Parole Chiave

All'esame, spesso una singola parola o frase nella domanda ti porta direttamente alla risposta corretta. Memorizza queste associazioni visive:

"Disaccoppiare" (Decouple) 👉 Amazon SQS
"Senza gestire server" 👉 Lambda o Fargate
"Distribuzione Globale / Cache" 👉 CloudFront
"Modelli / Infrastructure as Code" 👉 CloudFormation
"Storico API / Auditing" 👉 CloudTrail
"File condivisi tra EC2 Linux" 👉 Amazon EFS
"Data Warehouse / Analisi dati" 👉 Amazon Redshift
"Previsione di spesa" 👉 AWS Cost Explorer

✅ Strategia di Eliminazione

  • In quasi tutte le domande a risposta multipla, due opzioni sono sempre sbagliate o non esistono nemmeno come servizi reali. Concentrati sulle due rimanenti.
  • Se la domanda chiede la soluzione "più economica" (cost-effective) per lo storage, la risposta è quasi sempre S3 Glacier o S3 Standard-IA.
  • Se la domanda parla di ambiente "on-premise" o "ibrido", cerca servizi come Direct Connect, Storage Gateway o AWS Outposts.

⚠️ Trappole Comuni (Attenzione!)

  • CloudTrail vs CloudWatch: CloudTrail registra "CHI ha fatto un'azione". CloudWatch monitora "COME stanno performando le risorse" (CPU, RAM). Non confonderli!
  • Security Group vs NACL: Il Security Group protegge la singola Istanza (è stateful). La NACL protegge l'intera Subnet (è stateless).
  • S3 vs EBS: S3 è per file distribuiti (immagini, backup). EBS è il disco rigido collegato fisicamente a un'istanza EC2.

⏳ Gestione del Tempo

  • Hai 90 minuti per 65 domande. Sono quasi 1 minuto e 20 secondi a domanda. È un tempo molto abbondante, non correre!
  • Non farti prendere dal panico se non sai una risposta. Usa il tasto "Flag for Review" (Segna per la revisione), metti una risposta a intuito e vai avanti. Potrai tornarci alla fine.
  • Molto spesso, le domande successive contengono suggerimenti o ti ricordano servizi che ti aiutano a rispondere alle domande precedenti che avevi saltato.

7. Quiz di Preparazione

Mettiti alla prova con le domande ufficiali e pratiche. Clicca su "Mostra Risposta" per verificare la tua preparazione!

🗄️ Database

Quali servizi di database AWS supportano le applicazioni NoSQL?

Mostra Risposta

Risposte Corrette: Amazon DynamoDB, Amazon Neptune

Spiegazione: DynamoDB è progettato per dati chiave-valore e documentali; Neptune è progettato specificamente per i database a grafi (entrambi NoSQL).

🌐 Networking & Sicurezza (VPC)

Quale opzione di connettività per un VPC consente a una sottorete privata di connettersi a Internet?

Mostra Risposta

Risposta Corretta: Gateway NAT (NAT Gateway)

Spiegazione: Il NAT permette al traffico in uscita dalle istanze private di raggiungere Internet (ad es. per aggiornamenti software), bloccando però le connessioni dirette in entrata.

Un ingegnere di rete deve acquisire informazioni sul traffico IP in entrata e in uscita dal VPC. Quale funzionalità utilizzare?

Mostra Risposta

Risposta Corretta: Log di flusso VPC (VPC Flow Logs)

Spiegazione: I log di flusso registrano i pacchetti IP in entrata e in uscita, utilissimi per il troubleshooting e l'analisi della sicurezza (es. per vedere quale traffico è stato bloccato).

Che tipo di regole può aggiungere un ingegnere alle tabelle di una ACL di rete (NACL)?

Mostra Risposta

Risposta Corretta: Regole di autorizzazione e negazione (Allow e Deny)

Spiegazione: A differenza dei Security Group (che supportano solo regole Allow), le NACL operano a livello di subnet e permettono di negare esplicitamente il traffico da un IP specifico.

Quali affermazioni descrivono correttamente i gruppi di sicurezza (Security Groups)?

Mostra Risposta

Risposte Corrette:
1. Dispongono di una tabella in entrata e una in uscita.
2. Il gruppo predefinito consente tutto il traffico tra le risorse interne.

Spiegazione: I Security Group sono firewall stateful che operano a livello di istanza.

Cos'è un segmento di rete logico in un VPC che può esistere solo in una singola zona di disponibilità?

Mostra Risposta

Risposta Corretta: Sottorete (Subnet)

Spiegazione: Una subnet non può mai estendersi su più di una singola AZ.

💻 Compute e Storage

Quale affermazione relativa ad Amazon Machine Images (AMI) è vera?

Mostra Risposta

Risposta Corretta: Un'AMI è ancorata a livello di Regione.

Spiegazione: Le AMI sono specifiche per regione. Se vuoi lanciare la stessa istanza in un'altra regione, devi prima copiare l'AMI.

Quale soluzione fornisce archiviazione temporanea per un'istanza Amazon EC2?

Mostra Risposta

Risposta Corretta: Archivio dell'istanza (Instance Store)

Spiegazione: Si tratta di dischi fisicamente collegati all'host su cui gira EC2. Sono velocissimi ma temporanei (i dati si perdono se l'istanza viene fermata).

Quale tipo di volume EBS offre le prestazioni più elevate per le operazioni di lettura/scrittura frequenti?

Mostra Risposta

Risposta Corretta: SSD con capacità di IOPS allocata (io2)

Spiegazione: I volumi "io" (Provisioned IOPS) sono pensati per carichi di lavoro mission-critical o database che necessitano di prestazioni I/O costanti e garantite.

Quale classe di archiviazione S3 usare quando il modello di accesso ai dati è sconosciuto o cambia?

Mostra Risposta

Risposta Corretta: S3 Intelligent-Tiering

Spiegazione: Questa classe sposta automaticamente gli oggetti tra i livelli di accesso frequente e non frequente, ottimizzando i costi senza penalità per il recupero dati.

Quanto tempo è necessario per eseguire un recupero dei dati standard da Amazon S3 Glacier?

Mostra Risposta

Risposta Corretta: 3-5 ore

Spiegazione: Glacier è progettato per l'archiviazione a lungo termine ("a freddo"). Il recupero standard richiede qualche ora. Non è istantaneo come S3 Standard.

🛠️ Monitoraggio, Eventi e Auditing

Quale funzionalità monitora un parametro e invia un avviso quando il parametro cambia e supera una soglia?

Mostra Risposta

Risposta Corretta: Allarmi di Amazon CloudWatch

Spiegazione: Gli allarmi scattano al superamento delle soglie impostate per specifiche metriche (es. CPU > 80%).

Quale servizio può essere utilizzato con AWS CloudTrail per archiviare a lungo termine i log delle chiamate API?

Mostra Risposta

Risposta Corretta: Amazon Simple Storage Service (Amazon S3)

Spiegazione: CloudTrail registra le attività dell'account e può salvare i file di log in un bucket S3 per sicurezza e compliance.

Su quale tipo di log possono essere eseguite direttamente query con Amazon Athena?

Mostra Risposta

Risposta Corretta: Log di AWS CloudTrail (salvati su S3)

Spiegazione: Athena permette di interrogare file e log presenti nei bucket S3 usando normali query SQL.

⚙️ Automazione, System Administration & Scripting (Extra)

Quale servizio AWS crea un modello simile a script che rappresenta uno stack di risorse per fare il provisioning dell'infrastruttura?

Mostra Risposta

Risposta Corretta: AWS CloudFormation

Spiegazione: Fondamento del concetto di "Infrastructure as Code" (IaC) su AWS. Usa template JSON o YAML.

Un amministratore riceve un errore di Access Denied cercando di salvare un modello CloudFormation in S3. Qual è il motivo probabile?

Mostra Risposta

Risposta Corretta: L'amministratore (o il ruolo IAM usato) non ha le autorizzazioni per scrivere nel bucket (es. manca l'azione s3:PutObject).

Quale parte del comando (di apt-get) installa effettivamente i pacchetti aggiornati nel sistema operativo Linux Debian/Ubuntu?

Mostra Risposta

Risposta Corretta: apt-get upgrade

Spiegazione: "update" aggiorna solo la lista dei pacchetti disponibili; "upgrade" esegue il vero e proprio aggiornamento del software sul server. (Conoscenza utile lato OS).

Quale funzione in Python 3.x è l'alternativa raccomandata alla obsoleta funzione os.system() per eseguire comandi di sistema?

Mostra Risposta

Risposta Corretta: subprocess.run()

Spiegazione: Il modulo subprocess è il modo moderno, flessibile e sicuro per spawnare nuovi processi o eseguire comandi tramite script Python.

In ottica di "Principle of Least Privilege" (Principio del privilegio minimo), come dovrebbero essere assegnati i permessi IAM?

Mostra Risposta

Risposta Corretta: Assegnare solo le autorizzazioni strettamente necessarie per svolgere una specifica mansione, e niente di più.

Spiegazione: È la best practice fondamentale della sicurezza AWS per limitare l'esposizione al rischio.
